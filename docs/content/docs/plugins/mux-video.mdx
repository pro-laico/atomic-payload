`@pro-laico/mux-video` lets editors upload videos straight from the Payload admin and place them on pages with a ready-made video block. Uploads go to [Mux](https://www.mux.com/), which handles the encoding and adaptive streaming, and the bundled block renders the Mux player on your site, with controls for autoplay, looping, muting, preload, and a blurred poster frame.

<Callout type="info">
  `@pro-laico/mux-video` is a Tool package, a building block of the Atomic Payload stack meant to be used alongside `@pro-laico/core` and the other `@pro-laico/*` packages (most easily via the [atomic-payload template](/docs/templates/atomic-payload)), not on its own.
</Callout>

## Setup

<Tabs items={['Standalone', 'With the atomic-payload template']}>

<Tab value="Standalone">

<Callout type="info">
  `@pro-laico/core` comes bundled with this plugin — it's a dependency, not a separate install — and provides the shared fields, hooks, and utilities this plugin builds on. See [@pro-laico/core → Setup](/docs/plugins/core#setup).
</Callout>

Adding Mux video to your own Payload + Next.js project.

<Steps>

<Step>
### Add the plugin to your Payload config

```ts
import { buildConfig } from 'payload'
import { muxVideoPlugin } from '@pro-laico/mux-video'

export default buildConfig({
  plugins: [muxVideoPlugin()],
})
```

This registers the **`mux-video`** collection (in the admin's **Assets** group) where uploaded videos live, and applies the upstream Mux plugin's fields to it. With your Mux credentials in place (see [environment variables](#environment-variables)), editors can upload a video and Mux takes over encoding and streaming.
</Step>

<Step>
### Set your Mux credentials

The plugin reads your Mux API token and webhook secret from the [environment](#environment-variables) before `buildConfig` runs. Without `MUX_TOKEN_ID` and `MUX_TOKEN_SECRET`, the plugin logs a warning and uploads will fail, so set them before you start the app.
</Step>

<Step>
### Add the video block to your pages

The package ships a **`VideoChild`** block that lets editors pick an uploaded Mux video and set playback options (autoplay, loop, muted, preload, plus a blurred poster frame). Register it through `@pro-laico/atomic`'s child-blocks plugin so it renders inside your Atomic pages:

```ts
import { buildConfig } from 'payload'
import { childBlocksPlugin } from '@pro-laico/atomic/children'
import { Video } from '@pro-laico/mux-video/blocks/videoChild'

export default buildConfig({
  plugins: [
    childBlocksPlugin({ childBlocks: [Video] }),
    // ...other plugins
  ],
})
```

<Callout type="info">
  The block is **not** re-exported from the package root. Import the prebuilt `Video` block (or the `createVideoBlock` factory) from the `@pro-laico/mux-video/blocks/videoChild` subpath.
</Callout>
</Step>

<Step>
### Render it on the frontend

You don't mount the player by hand. The block's renderer is registered with `@pro-laico/atomic`'s child renderer, so once a page's blocks pass through `RenderChildren`, any `VideoChild` block automatically renders the Mux player:

```tsx
// app/(frontend)/[...slug]/page.tsx
import { RenderChildren } from '@pro-laico/atomic/children/render'

export default async function Page({ page }) {
  return (
    <main>
      <RenderChildren blocks={page.children} />
    </main>
  )
}
```

Need the player component directly for a custom layout? It's available at `@pro-laico/mux-video/blocks/videoChild/component` as `VideoChild`, which mounts `@mux/mux-video-react` with the block's settings.
</Step>

</Steps>

</Tab>

<Tab value="With the atomic-payload template">

The [atomic-payload template](/docs/templates/atomic-payload) already wires everything up: the plugin is in the config, the `VideoChild` block is registered with the child-blocks plugin (and given a styling field), and pages render through the Atomic child renderer. You just add your Mux credentials and use the block.

<Steps>

<Step>
### Add your Mux credentials

Copy the Mux keys into your `.env` (the template's `.env.example` lists them):

```plaintext
# .env (Mux credentials)
MUX_TOKEN_ID=
MUX_TOKEN_SECRET=
MUX_WEBHOOK_SIGNING_SECRET=
```

See [environment variables](#environment-variables) for what each one is for.
</Step>

<Step>
### Upload a video

In the admin's **Assets** group, open the **mux-video** collection and upload a video file. Mux handles encoding and streaming once it's uploaded.
</Step>

<Step>
### Drop the video block on a page

Editing a page, add a **Video** block, pick the uploaded video, and set the playback options (autoplay, loop, muted, preload, and the blurred poster frame). The template renders it through the Atomic child renderer, so the Mux player shows up on the frontend with no extra wiring.
</Step>

</Steps>

</Tab>

</Tabs>

## Options

`muxVideoPlugin(options?)` accepts:

<TypeTable
  type={{
    enabled: { type: 'boolean', default: 'true', description: 'Set false to turn the plugin off (it becomes a no-op).' },
    includeCollection: { type: 'boolean', default: 'true', description: 'Registers the bundled mux-video collection that holds uploaded videos. Set false if you register your own collection with the same slug.' },
    collectionOptions: { type: 'Partial<CollectionConfig>', description: "Tweaks merged onto the mux-video collection. Top-level keys replace, but `access` and `admin` are deep-merged, `fields` are appended, and `hooks` are merged per phase, so a partial override can't silently drop the collection's access rules." },
    adminThumbnail: { type: "'image' | 'gif' | 'none'", default: "'image'", description: 'What to show as the thumbnail in the admin list view.' },
    uploadSettings: { type: '{ cors_origin: string }', default: 'NEXT_PUBLIC_SERVER_URL', description: 'The CORS origin allowed to upload to Mux. Defaults to your site URL. See its keys below.' },
    initSettings: { type: '{ tokenId, tokenSecret, webhookSecret }', default: 'from env', description: 'Your Mux credentials. Defaults to reading the MUX_* environment variables; pass them explicitly to override. See its keys below.' },
  }}
/>

All options at their defaults, as a working starting point:

```ts
muxVideoPlugin({
  enabled: true,
  includeCollection: true,
  adminThumbnail: 'image',
  // collectionOptions is unset by default
  uploadSettings: {
    cors_origin: process.env.NEXT_PUBLIC_SERVER_URL || 'http://localhost:3000',
  },
  initSettings: {
    tokenId: process.env.MUX_TOKEN_ID || '',
    tokenSecret: process.env.MUX_TOKEN_SECRET || '',
    webhookSecret: process.env.MUX_WEBHOOK_SIGNING_SECRET || '',
  },
})
```

## Environment variables

The plugin picks these up automatically before `buildConfig` runs:

| Variable | Purpose |
| --- | --- |
| `MUX_TOKEN_ID` | Your Mux API access token ID. Required for uploads and Mux API calls. |
| `MUX_TOKEN_SECRET` | Your Mux API access token secret. Required alongside the token ID. |
| `MUX_WEBHOOK_SIGNING_SECRET` | Verifies incoming Mux webhooks (e.g. when an upload finishes encoding). |
| `NEXT_PUBLIC_SERVER_URL` | Used as the `cors_origin` for Mux uploads. Falls back to `http://localhost:3000`. |
