**Atomic Payload** is a website builder built on [Payload CMS](https://payloadcms.com/) that draws a hard line between front-end development and the backend. Editors build their entire website directly in Payload's admin dashboard — without ever having to touch real code. That separation makes it easy to enforce best practices and achieve great performance out of the box.

The building blocks come from a family of composable `@pro-laico/*` plugins: Tailwind styles written in the admin, recursive atomic blocks with copy-paste-duplicate, UnoCSS shortcuts, design tokens and swappable design sets, interactive actions, and custom forms with sanitation, validation, and rate limiting — plus images, fonts, icons, Mux video, and tracking.

<Callout type="info">
  Atomic Payload is not affiliated with Payload CMS in any capacity.
</Callout>

## How it's built

Every feature ships as an independently-installable Payload plugin (`sitePlugin`, `stylesPlugin`, `iconsPlugin`, `fontsPlugin`, `trackingPlugin`, and more), all sitting on top of the **`@pro-laico/core`** kernel. Core provides the shared type machinery — the single `PayloadAugment` interface plus the `Get<>` and `ExtractOrDefault` helpers — so each package defines its schema as stubs that resolve to your project's concrete shapes once you generate types. The whole thing is distributed both as a full starter template (Payload + Next.js + Tailwind that uses every plugin) and as the individual plugins you can add to an existing project one at a time.
