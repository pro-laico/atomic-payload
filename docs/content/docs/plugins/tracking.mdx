`@pro-laico/tracking` turns analytics on and off from the Payload admin. Enable PostHog, Google Tag Manager, or Vercel Analytics with a toggle and add the keys right in the dashboard — no juggling environment variables across every deployment.

<Callout type="info">
  `next`, `payload`, and `react` are required peers you already have in a Payload + Next.js app. The provider SDKs are **optional** — install only the ones you turn on.
</Callout>

Optional peer dependencies:

| Peer | Needed when |
| --- | --- |
| `posthog-js` | PostHog is enabled |
| `@next/third-parties` | Google Tag Manager is enabled |
| `@vercel/analytics` | Vercel Analytics is enabled |

## Setup

Add the plugin to your Payload config:

```ts
import { trackingPlugin } from '@pro-laico/tracking'
import { buildConfig } from 'payload'

export default buildConfig({
  plugins: [trackingPlugin()],
})
```

Then mount the composite provider at your app root, passing in the `Tracking` global. Without it, every provider short-circuits and nothing loads, so fetch it server-side and thread it down:

```tsx
// app/(frontend)/layout.tsx — a Server Component
import config from '@payload-config'
import { TrackingProvider } from '@pro-laico/tracking/provider'
import { getPayload } from 'payload'

export default async function RootLayout({ children }: { children: React.ReactNode }) {
  const payload = await getPayload({ config })
  const tracking = await payload.findGlobal({ slug: 'tracking' })

  return <TrackingProvider tracking={tracking}>{children}</TrackingProvider>
}
```

## Exports

**Subpath imports**

| Subpath | What's there |
| --- | --- |
| `@pro-laico/tracking` | The plugin, `Tracking` global, and provider field groups |
| `@pro-laico/tracking/provider` | The React providers (client) |
| `@pro-laico/tracking/schema` | Payload `Tracking` type augmentation stub |

**By kind**

- **Globals** — `Tracking`
- **Providers** (from `/provider`) — `TrackingProvider`, `PostHogProvider`, `GoogleTagManagerProvider`, `VercelProvider`
- **Fields** — `postHogGroup()`, `googleTagManagerGroup()` — drop-in provider field groups for composing your own Tracking global

<Callout type="info">
  The providers live behind `/provider` because they're client components that pull in the optional SDKs, which aren't safe to load on the server. Keeping them off the main barrel means the server-resolved entry (e.g. for `payload generate:importmap`) never reaches them.
</Callout>
