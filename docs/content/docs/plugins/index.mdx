Atomic Payload is a family of focused plugins for Payload CMS, each published under `@pro-laico/*`. Every one adds a single capability to your admin and your site, and most run happily on their own — so you can pull in just the pieces you want and leave the rest. The full [template](/docs/templates/atomic-payload) is all of them wired together.

## The plugin shape

Every plugin follows the same factory shape: `(opts) => (config) => config`, exported as both the default export and a named export. You add it to the `plugins` array in your Payload config:

```ts
import { stylesPlugin } from '@pro-laico/styles'
import { buildConfig } from 'payload'

export default buildConfig({
  plugins: [stylesPlugin({ /* options */ })],
})
```

Raw collections, globals, fields, hooks, and components are also exported as named imports for advanced composition. See each plugin's page for its options and exports, or [Add to an existing project](/docs/getting-started/existing-project) for the end-to-end flow.

## Packages

The sidebar groups these by role: **Foundation** is `core` (the shared kernel everything else builds on), **Standalones** run usefully on their own in any Payload project, and **Tools** are the supporting pieces the template composes. Most capabilities also have a friendlier guide under [Features](/docs/features/atomic-blocks); `richtext`, `zap`, and `core` are developer tooling, so the reference pages below are their only home.
