Manages custom fonts in Payload and ships them to disk for `next/font/local`. It registers an upload collection (`Font`) for woff/woff2/ttf/otf files, optionally registers a `fontSet` global picking the active sans/serif/mono/display fonts, and ships a build-time CLI (`atomic-fonts-download`) that pulls the active font files out of storage and writes them to disk where `next/font/local` can pick them up.

## Setup

Add the plugin to your Payload config:

```ts
import { buildConfig } from 'payload'
import { fontsPlugin } from '@pro-laico/fonts'

// With @pro-laico/styles ‚Äî fonts come from the active designSet's `font` group:
export default buildConfig({ plugins: [fontsPlugin()] })

// Standalone ‚Äî register the `fontSet` global to pick the active fonts:
export default buildConfig({ plugins: [fontsPlugin({ global: true })] })
```

## Options

`fontsPlugin(options?)` accepts:

<TypeTable
  type={{
    enabled: { type: 'boolean', default: 'true', description: 'Set false to no-op the plugin.' },
    fontOverride: { type: 'Partial<CollectionConfig>', description: 'Deep-merged onto the Font collection ‚Äî upload, access, and hooks merge onto the base (e.g. upload.staticDir keeps the base mimeTypes); fields replaces wholesale.' },
    global: { type: 'boolean | Partial<GlobalConfig>', default: 'false', description: 'true registers the standalone fontSet global; pass a partial GlobalConfig to override. fields are appended to the base font slots and access is deep-merged; other top-level keys replace.' },
  }}
/>

## Exports

**Subpath imports**

| Subpath | What's there |
| --- | --- |
| `@pro-laico/fonts` | The plugin, the `Font` collection, font upload fields, and the `fontSet` global |
| `@pro-laico/fonts/schema` | `Font` / `FontSet` schema stubs |
| `@pro-laico/fonts/scripts/downloadFonts` | `runDownloadFonts` ‚Äî the programmatic download entry |

**By kind**

- **Plugin** ‚Äî `fontsPlugin` (default)
- **Collections** ‚Äî `Font` ‚Äî the upload collection config
- **Globals** ‚Äî `FontSet`, `createFontSetGlobal`, `FONT_SET_SLUG` ‚Äî the standalone font-selection global
- **Fields** ‚Äî `fontUploadField` (the `font` group field for the designSet's Fonts tab), `fontUploadFields` (the four upload slots shared by the group and the global)
- **Scripts** (from `/scripts/downloadFonts`) ‚Äî `runDownloadFonts` ‚Äî selection resolution + storage-agnostic download + definition generation
- **Bin** ‚Äî `atomic-fonts-download` ‚Äî the published CLI (requires the package to be built)

## Environment variables

The download CLI reads the active font selection and resolves binaries using:

| Variable | Purpose |
| --- | --- |
| `LIVE_SITE_URL` | Site to read the active designSet / `fontSet` global from, and the base for resolving relative (local-storage) font URLs. |
| `SCRIPT_USER_EMAIL` | Payload user to authenticate as (needed to read auth-gated fonts and, for local storage, the file route). |
| `SCRIPT_USER_PASSWORD` | Password for that user. |

No storage token is required ‚Äî the downloader uses whatever URL Payload reports for each font.
