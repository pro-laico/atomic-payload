`@pro-laico/mux-video` brings video to your site, powered by [Mux](https://www.mux.com/). Upload videos through the Payload admin, let Mux handle encoding and adaptive streaming, and drop them onto your pages with a ready-made video block.

## Installation

<Tabs groupId="package-manager" items={['pnpm', 'npm', 'yarn']}>

<Tab value="pnpm">

```bash
pnpm add @pro-laico/mux-video
```

</Tab>

<Tab value="npm">

```bash
npm install @pro-laico/mux-video
```

</Tab>

<Tab value="yarn">

```bash
yarn add @pro-laico/mux-video
```

</Tab>

</Tabs>

<Callout type="info">
  `next`, `payload`, and `react` are required peers you already have in a Payload + Next.js app. `@oversightstudio/mux-video` is the upstream plugin this package wraps — install it too. The Mux player itself, `@mux/mux-video-react`, ships as a bundled dependency, so you don't install it separately.
</Callout>

Peer dependencies:

| Peer | Range | Needed when |
| --- | --- | --- |
| `@oversightstudio/mux-video` | `>=1.0.0` | Always — this package composes the upstream plugin |
| `next` | `>=15.0.0` | Always — already present in a Payload + Next.js app |
| `payload` | `>=3.0.0` | Always — already present in a Payload app |
| `react` | `>=19.0.0` | Always — already present in a Next.js app |

## Setup

```ts
import { buildConfig } from 'payload'
import { muxVideoPlugin } from '@pro-laico/mux-video'

export default buildConfig({ plugins: [muxVideoPlugin()] })
```

## Exports

**Subpath imports**

| Subpath | What's there |
| --- | --- |
| `@pro-laico/mux-video` | The `muxVideoPlugin` and the bundled `mux-video` extension collection |
| `@pro-laico/mux-video/blocks/videoChild` | The `VideoChild` block — exports the `createVideoBlock` factory, the built `Video` block, and the `VideoBlockOptions` type (they are intentionally **not** re-exported from the root) |
| `@pro-laico/mux-video/blocks/videoChild/component` | Child block renderer (`VideoChild`), which mounts the `@mux/mux-video-react` player |

**By kind**

- **Plugin** — `muxVideoPlugin`
- **Collections** — the local `mux-video` extension collection
- **Blocks** (from `/blocks/videoChild`) — the `createVideoBlock` factory, the built `Video` block (slug/`interfaceName` `VideoChild`), and the `VideoBlockOptions` type

## Environment variables

All picked up automatically:

| Env | Purpose |
| --- | --- |
| `NEXT_PUBLIC_SERVER_URL` | Used as the `cors_origin` for Mux uploads. Falls back to `http://localhost:3000`. |
| `MUX_TOKEN_ID` | Mux API token ID. |
| `MUX_TOKEN_SECRET` | Mux API token secret. |
| `MUX_WEBHOOK_SIGNING_SECRET` | Mux webhook signing secret. |
