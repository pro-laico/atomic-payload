title: "@pro-laico/fonts"

---

`@pro-laico/fonts` lets you manage custom fonts from the Payload admin and use them with `next/font/local`. Upload your font files, choose the active sans / serif / mono / display faces in the dashboard, and a build step pulls them into your app — no font files hardcoded in your repo.

## Setup

Add the plugin to your Payload config:

```ts
import { buildConfig } from 'payload'
import { fontsPlugin } from '@pro-laico/fonts'

// With @pro-laico/styles — fonts come from the active designSet's `font` group:
export default buildConfig({ plugins: [fontsPlugin()] })

// Standalone — register the `fontSet` global to pick the active fonts:
export default buildConfig({ plugins: [fontsPlugin({ includeFontSet: true })] })
```

## Options

`fontsPlugin(options?)` accepts:

<TypeTable
  type={{
    enabled: { type: 'boolean', default: 'true', description: 'Set false to no-op the plugin.' },
    fontOptions: { type: 'Partial<CollectionConfig>', description: 'Deep-merged onto the Font collection — upload, access, and hooks merge onto the base (e.g. upload.staticDir keeps the base mimeTypes) and fields are appended.' },
    includeFontSet: { type: 'boolean', default: 'false', description: 'When true, registers the standalone fontSet global — the active sans/serif/mono/display selection.' },
    fontSetOptions: { type: 'Partial<GlobalConfig>', description: 'Merged onto the fontSet global when includeFontSet is true — fields are appended to the base font slots and access is deep-merged; other top-level keys replace.' },
  }}
/>

## Exports

**Subpath imports**

| Subpath | What's there |
| --- | --- |
| `@pro-laico/fonts` | The plugin, the `Font` collection, font upload fields, and the `fontSet` global |
| `@pro-laico/fonts/schema` | `Font` / `FontSet` schema stubs |
| `@pro-laico/fonts/scripts/downloadFonts` | `runDownloadFonts` — the programmatic download entry |

**By kind**

- **Plugin** — `fontsPlugin` (default)
- **Collections** — `Font` — the upload collection config
- **Globals** — `FontSet`, `createFontSetGlobal`, `FONT_SET_SLUG` — the standalone font-selection global
- **Fields** — `fontUploadField` (the `font` group field for the designSet's Fonts tab), `fontUploadFields` (the four upload slots shared by the group and the global)
- **Scripts** (from `/scripts/downloadFonts`) — `runDownloadFonts` — selection resolution + storage-agnostic download + definition generation
- **Bin** — `atomic-fonts-download` — the published CLI (requires the package to be built)

## Environment variables

The download CLI reads the active font selection and resolves binaries using:

| Variable | Purpose |
| --- | --- |
| `LIVE_SITE_URL` | Site to read the active designSet / `fontSet` global from, and the base for resolving relative (local-storage) font URLs. |
| `SCRIPT_USER_EMAIL` | Payload user to authenticate as (needed to read auth-gated fonts and, for local storage, the file route). |
| `SCRIPT_USER_PASSWORD` | Password for that user. |

No storage token is required — the downloader uses whatever URL Payload reports for each font.
