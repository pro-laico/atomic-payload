`@pro-laico/richtext` gives you a rich-text block for your pages. Editors get a ready-to-use [Lexical](https://lexical.dev/) editor for writing formatted content in the Payload admin, and the package ships a renderer that turns that content into clean React on the frontend. It bundles three pieces: the `RichTextChild` block, the `defaultLexical` editor preset, and a JSX renderer.

## Setup

This package isn't a `buildConfig` plugin you add to the `plugins` array. Instead it hands you the pieces to wire rich text into your project yourself: an editor preset for the whole admin, a block your editors can drop into a page, and a renderer for the frontend.

<Steps>

<Step>
### Set the default editor

`defaultLexical` is an opinionated Lexical preset (bold/italic/headings/lists/links and a fixed + inline toolbar) tuned for Atomic Payload. Set it as the `editor` in your Payload config so every rich-text field in the admin uses it:

```ts
import { buildConfig } from 'payload'
import { defaultLexical } from '@pro-laico/richtext/default-lexical'

export default buildConfig({
  editor: defaultLexical,
  // ...
})
```

Want internal links to point at collections other than `pages`? Build the preset with the `createDefaultLexical` factory and pass `enabledCollections`:

```ts
import { createDefaultLexical } from '@pro-laico/richtext/default-lexical'

const editor = createDefaultLexical({ enabledCollections: ['pages', 'posts'] })
```
</Step>

<Step>
### Add the rich-text block

`RichText` is a ready-made Payload block (slug `RichTextChild`) you can drop into any `blocks` field. It carries a **Content** tab with a required rich-text area:

```ts
import { RichText } from '@pro-laico/richtext'

const PageBlocks = {
  name: 'children',
  type: 'blocks',
  blocks: [RichText],
}
```

To add your own fields to the Content tab â€” for example a CSS class field â€” use the `createRichTextBlock` factory instead of the pre-built `RichText` value. Pass `prependFields` / `appendFields` (the `RichTextBlockOptions` type) and they're spread at the start / end of the tab:

```ts
import { createRichTextBlock } from '@pro-laico/richtext'

const RichTextWithClassName = createRichTextBlock({ prependFields: [ClassNameField()] })
```

<Callout type="info">
  Using the [`@pro-laico/atomic`](/docs/plugins/atomic) child-blocks plugin? It already includes `RichTextChild` as one of its default blocks and lets you weave fields in through its `blockFields` option â€” you don't register the block separately.
</Callout>
</Step>

<Step>
### Render it on the frontend

The package ships `RichTextChild`, a React component that renders the block's serialized Lexical content as JSX (it uses `@payloadcms/richtext-lexical/react` under the hood and resolves internal links to their `href`). It's a child-block renderer: the render pipeline hands it the `block` (whose `.richText` holds the serialized state) plus a `pt` pass-through, so you don't pass content into it by hand.

If you're using [`@pro-laico/atomic`](/docs/plugins/atomic), this is already wired up â€” `RichTextChild` is registered as the renderer for the `RichTextChild` block, so rendering a page's `children` through atomic's `RenderChildren` draws your rich text automatically:

```tsx
import { RenderChildren } from '@pro-laico/atomic/children/render'

export default function Page({ page }: { page: { children: any } }) {
  return (
    <main>
      <RenderChildren blocks={page.children} />
    </main>
  )
}
```

The component sets no styling of its own, so reach for Tailwind's typography (`prose`) classes â€” via a class field on the block or your own wrapper â€” to style the output.
</Step>

</Steps>
