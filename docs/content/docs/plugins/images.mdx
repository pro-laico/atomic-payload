---

`@pro-laico/images` gives your Payload admin a place to upload images and favicons. The image collection converts uploads to WebP and generates a standard set of responsive sizes, a separate favicons collection keeps favicons out of your main image library, and the package ships a favicon-picker field and an image block you can use elsewhere. It's a tool the [atomic-payload template](/docs/templates/atomic-payload) and other plugins bring along, but you can add it on its own too.

## Setup

<Tabs items={['Standalone', 'With the atomic-payload template']}>

<Tab value="Standalone">

Adding the image collections to your own Payload project.

<Steps>

<Step>
### Add the plugin to your Payload config

```ts
import { buildConfig } from 'payload'
import { imagesPlugin } from '@pro-laico/images'

export default buildConfig({
  plugins: [imagesPlugin({ enabled: true })],
})
```

This registers the **Images** collection (where you upload images — every upload is converted to WebP and given a set of responsive sizes) and the **Favicons** collection (a separate place for `.ico` favicons). Both appear under an **Assets** group in the admin sidebar. Pass `includeFavicons: false` if you only want the Images collection.
</Step>

<Step>
### Upload your images

Open the **Images** collection in the admin and upload your images. Each one gets an `alt` text field (required, used as the title) and a focal point. Accepted types are PNG, JPEG, WebP, and AVIF.
</Step>

<Step>
### Add blur placeholders (optional)

The plugin does **not** wire up blur placeholders itself — pnpm doesn't install the optional peer next to this package, so the plugin can't reach it. Install [`@oversightstudio/blur-data-urls`](https://www.npmjs.com/package/@oversightstudio/blur-data-urls) yourself and register it **after** `imagesPlugin`, passing the exported `Images` collection:

```ts
import { buildConfig } from 'payload'
import { imagesPlugin, Images } from '@pro-laico/images'
import { blurDataUrlsPlugin } from '@oversightstudio/blur-data-urls'

export default buildConfig({
  plugins: [
    imagesPlugin({ enabled: true }),
    blurDataUrlsPlugin({
      enabled: true,
      collections: [Images],
      blurOptions: { blur: 18, width: 32, height: 'auto' },
    }),
  ],
})
```

That generates a small blurred placeholder for each upload, which the [image block](#the-image-block) can show while the full image loads.
</Step>

</Steps>

</Tab>

<Tab value="With the atomic-payload template">

The [atomic-payload template](/docs/templates/atomic-payload) already includes `@pro-laico/images` — the **Images** and **Favicons** collections, the blur-placeholder plugin wired against the Images collection, and the image block registered with the other Atomic blocks. You just upload your assets.

<Steps>

<Step>
### Upload images and favicons

Open the **Images** collection and upload your images (each needs `alt` text), and the **Favicons** collection for any `.ico` favicons.
</Step>

<Step>
### Pick a favicon

In the **Site Metadata** global, choose the light and dark favicons for your site. Individual pages can override them in their SEO tab.
</Step>

</Steps>

</Tab>

## Using it in your app

### The favicon field

`FaviconField` is a ready-made upload field that points at the Favicons collection, so you can add a favicon picker to any global or collection. It's what [`@pro-laico/site`](/docs/plugins/site) uses for the site's favicons; add it to your own config the same way:

```ts
import { FaviconField } from '@pro-laico/images'

const SiteMetaData = {
  slug: 'siteMetaData',
  fields: [FaviconField({ name: 'lightFavicon' }), FaviconField({ name: 'darkFavicon' })],
}
```

Pass a `name` (and anything else a Payload upload field accepts) to mount more than one picker. The field always targets the `favicons` collection, so an override can't accidentally re-point it.

### The image block

The package ships an **Image** block (slug `ImageChild`) that lets editors drop an image into [Atomic](/docs/plugins/atomic) content. It isn't registered by `imagesPlugin` — it's one of the default child blocks the Atomic renderer wires up — so in most setups you get it for free. To register it in your own block list, import it from the `@pro-laico/images/blocks/imageChild` subpath:

```ts
import { createImageBlock, Image } from '@pro-laico/images/blocks/imageChild'

// The prebuilt block:
const blocks = [Image]

// …or build your own, prepending/appending fields (e.g. a class-name field):
const customImageBlock = createImageBlock({ prependFields: [], appendFields: [] })
```

The block lets the editor choose the image, a stored size, alt text, and `next/image` options (priority, blur, loading, and more). On the front end it renders through `@pro-laico/atomic/children`, which resolves the chosen image and outputs an optimized `next/image`.

<Callout type="info">
  The Image block is **not** re-exported from the package root — import `createImageBlock` or `Image` from the `@pro-laico/images/blocks/imageChild` subpath.
</Callout>
