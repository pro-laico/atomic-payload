`@pro-laico/styles` brings your design system into the Payload admin. Write Tailwind classes directly on your blocks and author colors, spacing, animations, reusable shortcuts, and entire swappable design sets as content. There's no `tailwind.config` to edit and no redeploy to restyle your site — saving in the admin regenerates the stylesheet your frontend serves.

</Callout>

## How the CSS reaches your app

The plugin registers two collections you edit in the admin — **`designSet`** (your theme: colors, spacing, animations, prose, and other tokens) and **`shortcutSet`** (reusable class groupings, like a Tailwind component class) — plus two hidden **CSS storage globals** (`draftStorage` and `publishedStorage`).

When you save a design set, shortcut set, or any document that carries authored classes, a `beforeChange` hook regenerates the stylesheet with UnoCSS and writes it into those storage globals. Your frontend reads the stored CSS and injects it into the page `<head>`, and reads the active design set for its `<html>` / `<body>` class names. Switching which design set is active swaps your whole look — no rebuild.

You wire the CSS hook one of two ways, and the Setup tabs below show both:

- **Standalone** — pass a `getCached` getter and the plugin attaches its own `cssHook`, so this package processes CSS entirely on its own.
- **With the template** — pass the project's `atomicHook` (from `@pro-laico/atomic`), which handles CSS along with the rest of the runtime.

## Setup

<Tabs items={['Standalone', 'With the atomic-payload template']}>

<Tab value="Standalone">

Adding styles to your own Payload + Next.js project, with no `@pro-laico/atomic`.

<Steps>

<Step>
### Add the plugin to your Payload config

`stylesPlugin` always needs a `generateLivePreviewPath` (it's shared by both collections for live preview). Pass a `getCached` getter to attach the standalone CSS hook so this package generates the stylesheet itself:

```ts
import { buildConfig } from 'payload'
import { stylesPlugin, type CssProcessorGetCached } from '@pro-laico/styles'

// Reads the active designSet / shortcutSet, the generated site-css, and the
// page atomic-classes — see the next step.
const getCached: CssProcessorGetCached = async (tag, draft) => {
  const mod = await import('@/cache/getCached')
  return mod.default(tag, draft)
}

export default buildConfig({
  plugins: [
    stylesPlugin({
      generateLivePreviewPath: yourPreviewUrl,
      getCached,
    }),
  ],
})
```

This registers the **`designSet`** and **`shortcutSet`** collections and the `draftStorage` / `publishedStorage` globals the generated CSS is written to.

<Callout type="info">
  Pairing this with [`@pro-laico/fonts`](/docs/plugins/fonts)? Pass `designSet: { fontField: fontUploadField() }` so the design set's **Fonts** tab gets the font picker — that's how the template wires the two together (see the other tab).
</Callout>
</Step>

<Step>
### Provide the `getCached` getter

The CSS processor needs a getter that resolves the active design set, the active shortcut set, the generated `site-css`, and the page `atomic-classes`. Build it from `@pro-laico/core`'s stock registry and only override what your project doesn't have (e.g. header / footer collections):

```ts
// src/cache/getCached.ts
import { cache } from 'react'
import configPromise from '@payload-config'
import 'server-only'
import { createGetCached, defaultGetRegistry, type GetRegistry } from '@pro-laico/core/cache'

const registry: GetRegistry = {
  ...defaultGetRegistry,
  header: (async () => undefined) as GetRegistry['header'],
  footer: (async () => undefined) as GetRegistry['footer'],
}

export default cache(createGetCached(configPromise, registry))
```

Each getter is wrapped with revalidation tags, so editing a design set or shortcut set in the admin flows through to your page (and to live preview).
</Step>

<Step>
### Let your blocks carry classes

The class textarea field — where you type Tailwind/UnoCSS classes — ships in this package. Add `ClassNameField` to any collection whose documents should feed the stylesheet (e.g. your `pages` blocks). When such a document is saved, the standalone hook collects every `*ClassName` value (including those nested in blocks), stores them, and regenerates the CSS:

```ts
import { ClassNameField } from '@pro-laico/styles/fields/className'

// inside a collection's fields:
ClassNameField({ namePrefix: 'main', defaultValue: 'mx-auto max-w-3xl px-6 py-12' })
```

Attach the same standalone hook to that collection's `beforeChange` so saving a page regenerates the stylesheet with its classes — `createCssHook(getCached)` builds it.
</Step>

<Step>
### Render the generated stylesheet on your frontend

Your frontend reads the generated CSS from the storage globals via `getCached('site-css', draft)` and injects it into `<head>`. Read the active design set too, for its `<html>` / `<body>` / wrapper class names:

```tsx
// app/(frontend)/layout.tsx
import { draftMode } from 'next/headers'

import getCached from '@/cache/getCached'

export default async function RootLayout({ children }: { children: React.ReactNode }) {
  const { isEnabled: draft } = await draftMode()

  const css = await getCached('site-css', draft)
  const ds = await getCached('designSet', draft)

  return (
    <html lang="en" className={ds?.htmlClassName ?? undefined}>
      <head>
        <style id="atomic-generated" type="text/css" dangerouslySetInnerHTML={{ __html: css || '' }} />
      </head>
      <body className={ds?.bodyClassName || undefined}>
        <div className={`${ds?.wrapperClassName ?? ''} isolate`}>{children}</div>
      </body>
    </html>
  )
}
```

Now picking a different design set in the admin restyles the whole site on the next request — the design set's tokens become the `var(--…)` values your classes resolve against.
</Step>

<Step>
### Author your design in the admin

Open the **`designSet`** collection and edit its tabs — colors, sizes, animations, prose, and other tokens — then mark one set active. Author reusable class groupings in the **`shortcutSet`** collection. Only one design set is active at a time, so you can keep alternates as drafts and switch the active set to swap the whole look.
</Step>

</Steps>

The [`styles-only` example](/docs/examples/styles-only) wires this exact standalone path end to end.

</Tab>

<Tab value="With the atomic-payload template">

The [atomic-payload template](/docs/templates/atomic-payload) already wires everything — the `designSet` + `shortcutSet` collections, the CSS hook (through `@pro-laico/atomic`'s `atomicHook`), the storage globals, and the layout that serves the generated stylesheet. The template uses the shared `atomicHook` instead of a `getCached` getter, and turns on the font picker:

```ts
// src/plugins/styles.ts
import { stylesPlugin } from '@pro-laico/styles'
import { fontUploadField } from '@pro-laico/fonts'
import { atomicHook } from '@pro-laico/atomic/hook'
import { generateLivePreviewPath } from '@pro-laico/core'

export const stylesPluginConfig = stylesPlugin({
  atomicHook,
  generateLivePreviewPath,
  designSet: { fontField: fontUploadField() },
  registerTypescriptSchema: false,
})
```

You just author your design and build.

<Steps>

<Step>
### Author your design set

Open the **`designSet`** collection and edit its tabs — colors, sizes, animations, prose, and other tokens. If you've also set up [`@pro-laico/fonts`](/docs/plugins/fonts), the **Fonts** tab is where you pick the active faces.
</Step>

<Step>
### Add reusable shortcuts

In the **`shortcutSet`** collection, author reusable UnoCSS shortcuts — a class grouping you name once and reuse anywhere. The built-in default shortcuts show as read-only rows for reference.
</Step>

<Step>
### Mark a set active and save

Mark one design set active and save. The `atomicHook` regenerates the stylesheet into the storage globals, and the template's frontend layout already serves it — reload to see the new design. Keep alternates as drafts and switch the active set to swap the whole look, no redeploy needed.
</Step>

</Steps>

</Tab>

## Options

`stylesPlugin(options)` accepts:

<TypeTable
  type={{
    generateLivePreviewPath: { type: '(args) => Promise<string> | string', description: 'Required. Builds the live-preview URL for the active set. Shared by both collections.' },
    atomicHook: { type: 'CollectionBeforeChangeHook', description: "Wire the project's all-in-one hook (from @pro-laico/atomic) to handle CSS — and forms / actions — for both collections." },
    getCached: { type: 'CssProcessorGetCached', description: 'Provide a getter for the active sets and stored classes to attach the standalone cssHook, so this package generates CSS on its own without @pro-laico/atomic.' },
    cssHookOptions: { type: 'CssHookOptions', description: 'Forwarded to the standalone cssHook when getCached is set — override the slugs and storage globals it reads and writes.' },
    designSet: { type: 'false | object', default: 'enabled', description: "The designSet collection. Pass false (or { enabled: false }) to skip it. Object keys: access, collection (merged onto the collection), and fontField (a Payload Field for the Fonts tab — e.g. @pro-laico/fonts' fontUploadField())." },
    shortcutSet: { type: 'false | object', default: 'enabled', description: 'The shortcutSet collection. Pass false (or { enabled: false }) to skip it. Object keys: access, collection (same as designSet), and defaultShortcuts (read-only rows shown in the admin).' },
    includeStorageGlobals: { type: 'boolean', default: 'true', description: 'Set false to skip registering the draft / published CSS storage globals.' },
    registerTypescriptSchema: { type: 'boolean', default: 'true', description: "Appends @pro-laico/zap's schema extensions to config.typescript.schema so payload generate:types resolves the design-set field references. Set false if you already wire that schema yourself (e.g. via @pro-laico/core)." },
    enabled: { type: 'boolean', default: 'true', description: 'Set false to turn the whole plugin off.' },
  }}
/>
